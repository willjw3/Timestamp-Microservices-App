API Project: Timestamp Microservice

Terms of Use :

  1. Our API endpoint is GET [project_url]/api/timestamp/:date_string
  2. A date string is valid if can be successfully parsed by new Date(date_string).
    Note that the unix timestamp needs to be an integer (not a string) specifying milliseconds.
    To ensure a UTC timestamp, only date strings compliant with ISO-8601 (e.g. "2016-11-20") will be accepted.
  3. Empty date strings will return a timestamp for the current date.
  4. Valid dates strings are returned as a JSON having the structure
    {"unix": <date.getTime()>, "utc" : <date.toUTCString()> }
    e.g. {"unix": 1479663089000 ,"utc": "Sun, 20 Nov 2016 17:31:29 GMT"}
  5. Invalid date strings will result in the following output:
    {"unix": null, "utc" : "Invalid Date" }.

Example Inputs

  • <user's chosen url>/api/timestamp/2015-12-25
  • <user's chosen url>/api/timestamp/1450137600

Example Output:

{"unix":1451001600000, "utc":"Fri, 25 Dec 2015 00:00:00 GMT"}
